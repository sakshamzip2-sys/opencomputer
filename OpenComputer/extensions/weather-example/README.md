# Weather Example

A reference example plugin generated by `opencomputer plugin new
weather-example --kind provider`. Ships as a **bundled extension** so
new contributors can read one self-contained, tested plugin before they
start authoring their own.

It intentionally does NOT call a real weather API — every prompt returns
the same canned reply so the plugin works offline with zero API keys
and zero network dependencies. Swap out the hardcoded string in
`provider.py` with your own backend call when using this as a starting
point.

## What it demonstrates

- The scaffolder output shape for `kind=provider` — what `plugin.json`,
  `plugin.py`, and `provider.py` look like when produced by the template.
- How a `BaseProvider` subclass implements `complete()` and
  `stream_complete()`.
- How `register(api)` registers the provider under a short name via
  `api.register_provider(...)`.
- How a plugin's own test suite asserts `register` is callable without
  importing any OpenComputer internals (see `tests/`).

## Files

| File | Purpose |
|---|---|
| `plugin.json` | Manifest — id, kind, entry module, profile scoping |
| `plugin.py` | Entry module called by the loader's `register(api)` |
| `provider.py` | `WeatherExampleProvider(BaseProvider)` implementation |
| `tests/test_weather_example.py` | Minimal smoke test for `register` |

## Configuration

None. The provider ignores every argument and returns a hardcoded
string. That's the whole point — it is a **demo**, not a production
plugin.

## Re-generating

This directory is the output of:

```bash
opencomputer plugin new weather-example \
    --kind provider \
    --path extensions \
    --description "Example provider plugin that returns hardcoded weather for demos" \
    --author "OpenComputer Contributors"
```

followed by light hand-editing of `provider.py` (to add the hardcoded
reply) and `plugin.json` (to add `"profiles": ["*"]`). If you change
the templates under `opencomputer/templates/plugin/provider/`, you can
regenerate this extension with `--force` to compare outputs.
